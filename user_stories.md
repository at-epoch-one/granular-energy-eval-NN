# User Stories
This section includes the user stories defined for the system during development with the Scrum methodology.

---

### **HdU1 – Configure neural network**
**Description:** As a user, I need to configure the layers of a neural network and the training parameters.  
**Priority:** High

---

### **HdU2 – Choose meter**
**Description:** As a user, I need to choose the meter that will be used to measure the consumption.  
**Priority:** Medium

---

### **HdU3 – Create project**
**Description:** As a user, I need to create a new project to organize and store different case studies.  
**Priority:** High

---

### **HdU4 – Create case study**
**Description:** As a user, I need to create a case study to execute the training measurement.  
**Priority:** High

---

### **HdU5 – View case studies**
**Description:** As a user, I need to access the projects and case studies I have created in order to edit or delete them.  
**Priority:** High

---

### **HdU6 – View training results**
**Description:** As a user, I need to know the values of the loss function during training to evaluate its evolution and other metrics such as the *f1 score* and *accuracy*.  
**Priority:** High

---

### **HdU7 – View energy consumption results**
**Description:** As a user, I need to know the energy consumption data of the model’s training to evaluate and optimize it.  
**Priority:** High

---

### **HdU8 – Control measurement execution**
**Description:** As a user, I need to be able to start and stop the measurement of a case study.  
**Priority:** High

---

### **HdU9 – View real-time execution status**
**Description:** As a user, I need to see the current status of the training and track its progress.  
**Priority:** High

---

### **HdU10 – Download error file**
**Description:** As a user, I need to be able to download the error file when an execution fails.  
**Priority:** Medium

---

### **HdU11 – Register**
**Description:** As a user, I need to be able to register in the tool to access it later.  
**Priority:** Low

---

### **HdU12 – Download results**
**Description:** As a user, I need to be able to download the training and energy consumption results in different formats to analyze them.  
**Priority:** Medium

---

## Administrator Stories



### **HdU13 – System access**
**Description:** As a user and administrator, I need to be able to log in with my credentials to access the functionalities allowed for my role.  
**Priority:** High

---

### **HdU14 – Delete users**
**Description:** As an administrator, I need to be able to delete users registered in the system.  
**Priority:** Medium

---

### **HdU15 – Modify user data**
**Description:** As an administrator, I need to be able to modify data such as a user's password in the system.  
**Priority:** Low

---

### **HdU16 – Control measurement executions**
**Description:** As an administrator, I need to control the execution of users’ measurements to start or stop them when necessary.  
**Priority:** High

---

### **HdU17 – Upload dataset**
**Description:** As a user and administrator, I need to be able to upload a dataset and have the system validate it for use in model training.  
**Priority:** Medium

---

### **HdU18 – Delete datasets**
**Description:** As an administrator, I need to be able to delete datasets stored in the system to free up memory.  
**Priority:** High

---

### **HdU19 – Modify user projects**
**Description:** As an administrator, I need to be able to view and delete users’ projects and case studies.  
**Priority:** Medium
